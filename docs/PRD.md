# 產品需求文件 (PRD) - 漫遊索引系統

## 1. 專案概述

### 背景與動機
在數位漫畫蓬勃發展的時代，作品散落在 LINE Webtoon、Kakao Webtoon、實體書等多個平台，讀者面臨「漫畫荒」與「管理難」的雙重困境。傳統平台僅以類別（如冒險、愛情）進行分類，難以觸及讀者核心的審美與情緒需求。本系統旨在打造一個結合 AI 深度學習與社群標籤機制的漫畫發現平台，作為讀者的「漫畫入口網站」。

### 目標用戶
1. **重度漫畫讀者**：已看完熱門大作，正在尋找小眾或冷門佳作的人。
2. **跨平台讀者**：同時追蹤多個不同漫畫平台，並需要統一管理閱讀清單的人。
3. **新手讀者**：想進入漫畫世界但面對龐大類別感到迷茫的受眾。

### 核心價值主張
- **精準媒合**：透過情緒標籤與畫風分析，突破傳統分類限制。
- **一站式管理**：整合散落各處的閱讀紀錄與進度提醒。
- **透明評價**：互動式避雷機制，減少時間成本，提升閱讀體驗。

## 2. 功能需求

| 功能名稱 | 使用者故事 (User Story) |
| :--- | :--- |
| **多維度 AI 情感推薦引擎** | 作為重度讀者，我希望系統能提供「致鬱」、「燃爆」等情緒標籤，以便我根據當下心情找到共鳴的作品。 |
| **「畫風」視覺檢索系統** | 作為重視視覺層次的讀者，我希望上傳喜歡的截圖即可找到風格相近的作品，以便我極速縮短挑選時間。 |
| **跨平台追蹤書櫃** | 作為跨平台讀者，我希望能在一個系統內查看所有平台的閱讀進度，以便我統一管理並接收更新提醒。 |
| **互動式「避雷」評論區** | 作為想避開爛尾作品的讀者，我希望能看到「虎頭蛇尾」等雷點標註，以便我在投入時間前進行過濾。 |
| **「漫話室」社群** | 作為漫畫愛好者，我希望能加入特定作品的討論區並分享推薦書單，以便與同好交流並發現更多優質作品。 |

## 3. 非功能需求

### 技術限制
- **後端框架**：Flask (Python)
- **前端模板**：Jinja2 + Vanilla CSS (Aesthetics prioritized)
- **資料庫**：SQLite (用於儲存用戶、書櫃進度、標籤與評論)
- **開發工具**：Antigravity AI Agent

### 效能與安全考量
- **響應速度**：畫風檢索與 AI 推薦應在 3 秒內給出初步結果（MVP 階段可先以模擬數據或輕量化模型實作）。
- **數據隱私**：用戶跨平台導入的資料需安全存儲，且僅供該用戶查看。
- **SEO 優化**：確保系統具備良好的標題與描述，方便讀者透過搜尋引擎找到平台。

---

## 4. MVP 範圍 (最小可行性產品)

### Must Have (必要功能)
- 用戶註冊與登入系統。
- 基礎漫畫資料庫與手動「跨平台書櫃」管理功能。
- 基礎的情緒標籤分類與篩選。
- 帶有「雷點標籤」的評論系統。

### Should Have (重要功能)
- 「畫風」視覺檢索系統（基礎特徵比對）。
- 作品更新提醒通知。
- 推薦書單（Collection）建立與公開分享。

### Nice to Have (加分功能)
- 自動同步第三方平台的閱讀進度。
- 「漫話室」限時討論區的自動生成。
- 深度學習驅動的個性化閱讀報告。
