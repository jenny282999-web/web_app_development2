# 產品需求文件 (PRD) - 漫畫推薦系統

## 1. 專案概述

### 背景與動機
在漫畫愛好者的日常生活中，經常會遇到「漫荒」的情況——即看完當前追蹤的漫畫後，不知道接下來該看什麼。市場上雖然有許多漫畫平台，但缺乏一個專注於「推薦」與「社群互動」的整合系統，能幫助用戶精準找到符合口味的新作。

### 目標用戶
- 不分種族、性別、年齡，所有喜歡看漫畫的人。
- 正在尋找新漫畫、對特定類別有偏好的重度或輕度讀者。

### 核心價值主張
提供多維度的分類篩選與即時排行榜，結合章節評論社群，並提供靈活的閱讀體驗（下拉式/翻頁式），讓尋找與閱讀漫畫變得更簡單、更有趣。

---

## 2. 功能需求

### 2.1 漫畫分類系統
- **使用者故事**：作為一名讀者，我希望可以依照語言、書本內容（題材）、性別傾向進行分類篩選，以便我能快速縮小範圍找到感興趣的書。
- **功能細節**：
    - 語言：中文、日文、英文等。
    - 內容：冒險、戀愛、科幻、懸疑等。
    - 性別傾向：少年向、少女向、青年向等。

### 2.2 類別推薦排行榜
- **使用者故事**：作為一名讀者，我希望看到各個類別的熱門排行榜，以便了解當前最受歡迎的作品。
- **功能細節**：首頁或分類頁顯示「熱門 Top 10」或「評分最高」列表。

### 2.3 全域書名搜尋
- **使用者故事**：作為一名讀者，我希望網頁最上方有一個搜尋欄，以便我能直接輸入書名找到特定的漫畫。
- **功能細節**：固定在導覽列（Navbar）的搜尋框，支援關鍵字即時檢索。

### 2.4 章節留言區
- **使用者故事**：作為一名讀者，我希望在每看完一個章節後能在留言區發表意見，以便與其他讀者交流心得。
- **功能細節**：漫畫閱讀頁面章節末尾設有評論框，支援顯示其他用戶的留言。

### 2.5 彈性閱讀模式
- **使用者故事**：作為一名讀者，我希望在看漫畫時可以自由切換下拉式（卷軸）或翻頁式，以便符合我在不同裝置上的閱讀習慣。
- **功能細節**：閱讀器介面提供切換按鈕，切換時即時變更顯示邏輯。

---

## 3. 非功能需求

### 技術限制
- **後端**：使用 Flask 框架。
- **前端**：使用 HTML + Jinja2 模板引擎 + Vanilla CSS。
- **資料庫**：使用 SQLite 儲存漫畫資訊、用戶留言與排行榜數據。

### 效能與安全考量
- 圖片加載需考慮流暢度。
- 留言功能需有基本的內容過濾，防止惡意洗板。

---

## 4. MVP 範圍

### Must Have (必須有)
- 基礎漫畫列表與詳細資訊頁。
- 語言/內容/性別傾向分類篩選。
- 書名搜尋功能。
- 基礎閱讀器（支援至少一種模式）。

### Should Have (應該有)
- 類別排行榜功能。
- 章節留言區（新增與顯示留言）。
- 閱讀模式切換（下拉式與翻頁式）。

### Nice to Have (以後有)
- 用戶註冊與登入（收藏喜愛的漫畫）。
- 留言點讚功能。
- AI 個人化推薦算法。
